g) How does C affect the bias and variance of your classifier?¶

In general, bias and variance are inversely related, meaning that when bias increases, variance decreases (with a less complex model), furthermore, explicit regularization is used to control mthe model complexity. Looking at the 'Classification Error vs. C Values' plot from (d), we can clearly see that the test error decreases until we reach a C* value of 0.10, which is the optimal value. If we increase our C value even further beyond that point, we can see that our model starts to overfit, whereas, in fact, the variance increases, and the model's bias decreases (Lecture 5: p. 9).

g) How does the prediction error in f) compare with the prediction error in d)? Why?¶

The prediction error of the linear regression model in (f) is 3286.3379, whereas the prediction error of the lasso regression in (d) is at its minimum, 3191.6960. One of the differences between the models is that we perform an extensive search for an ideal regularization parameter in (d), while for (f), we just use the best values selected in the lasso regression model. Therefore, it could be, without any regularization, that there exists a slight overfitting in the linear regression model, or that if we were to apply a forward search algorithm to select the ideal features for the linear regression, the model might choose different columns, and its testing error could decrease (converge to lasso regression).

h) How does the regularization parameter affect bias and variance.¶

A larger regularization parameter incurs larger bias and smaller variance, while a smaller regularization parameter incurs smaller bias and larger variance. This tradeoff between bias and variance is fundamental in machine learning models. It iss precisely why we observe a U-shaped curve in (d), illustrating how the relationship between bias and variance can impact the model's performance.

e) How does the prediction error in d) compare with the prediction error in b)? Why?¶

Using the 34 selected variables from the trained model in (b), the logistic regression model leads to a prediction error of 0.080934 in (d), whereas the regularized logistic lasso regression has a classification error of 0.085823 in (b). Since the model's inherent linearity already has a low level of complexity, we cannot gain as much as one would think by applying the regularization procedure. Another source of noise to be mentioned in (b) is the classification of the predicted response variables, where I used (-infinity, 0] to classify a value as 0, otherwise 1. Looking at my classification rule analysis (plot and numbers), we can clearly see that the model slightly favors 0's since we included that value in the left interval. That being said, the standard logistic regression model slightly outperforms the regularization model.

In k-fold cross-validation, the data is divided into k parts, using k-1 parts to train the model and calculating the classification error on the kth part of the dataset (in this case, the training dataset). Since the training set has 2,760 rows, in our extreme case when k=10, the validation set is reduced to 276 data points. Additionally, one can increase the randomness by shuffling the data before applying k-fold cross-validation.

In parallel, on the second plot, the test error remains the same for all values of k except for k=2, where a different value of C generates the optimal case. Another interesting fact to mention is that the cross-validation errors are smaller than the test error. This is because cross-validation still tends to overfit the data.
